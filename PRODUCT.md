# Clippi 제품 정의

## 제품이 해결하는 문제
Clippi는 사용자가 여러 플랫폼에서 발견한 링크를 저장한 뒤, 다시 찾고 소비하는 과정이 번거롭고 산만해지는 문제를 해결한다.

일반적인 북마크 앱은 "저장"까지만 돕고, 이후에는 사용자가 직접 제목을 정리하고 분류를 만들고 다시 찾아야 한다. 그 결과 북마크가 쌓이기만 하고 다시 소비되지 않는 경우가 많다. Clippi는 이 지점을 줄여서, 저장 직후부터 더 보기 좋고 덜 귀찮은 정리 경험을 제공하고, 궁극적으로는 저장한 북마크를 실제 활용으로 이어지게 만드는 것을 목표로 한다.

## 핵심 사용자 가치
- 링크를 빠르게 저장할 수 있다.
- 무료 AI 기능으로 저장 진입장벽이 낮아진다.
- 저장된 링크가 카테고리와 소분류 단위로 더 읽기 쉽게 정리된다.
- 메타데이터와 AI 요약을 통해 나중에 다시 봐도 무엇인지 바로 이해할 수 있다.
- 저장한 북마크를 잊지 않고 다시 꺼내보게 만든다.
- 유튜브 북마크는 예외적으로 더 편한 스와이프 재소비 UX를 제공한다.
- X/Twitter 링크도 일반 북마크보다 더 깨끗한 제목/이미지로 저장된다.

## 현재 구현 기준 제품 정체성
현재 Clippi의 정체성은 "AI로 저장 접근성을 낮추고, 북마크를 다시 소비하기 쉽게 만드는 프리미엄 북마크 앱"이다.

여기서 핵심은 두 가지다.
- 1차 핵심: 저장된 링크를 덜 수고롭게 정리하고 다시 찾게 만드는 것
- 2차 핵심: 저장한 북마크를 잊지 않고 다시 소비하게 만드는 것

무료 AI는 수익화 기능이 아니라, 다른 북마크 앱보다 더 쉽게 시작하게 만들고 "한 번 써보고 싶다"는 호기심을 만드는 핵심 온보딩 가치다.

즉, 지금 단계에서 Clippi는 "모든 SNS를 한 앱에서 소비하는 통합 뷰어"가 아니라, "저장과 재탐색 경험을 AI로 더 쉽게 만들고, 이후 활용 가능성까지 여는 북마크 앱"으로 정의하는 것이 맞다.

## 현재 코드 기준 포함 범위
- 카테고리 기반 홈 화면
- 카테고리별 북마크 목록
- 소분류 생성, 소분류 이동, 소분류별 그룹 표시
- 무료 AI 요약 제목 생성
- 무료 AI 소분류 추천
- URL 메타데이터 수집
- X/Twitter용 `fixupx.com` 기반 메타데이터 보정
- YouTube 전용 세로형 snackable viewer
- 공유 시트 기반 북마크 추가

## 공유 저장 UX 원칙
- 공유 저장은 Clippi의 핵심 진입 경로로 본다.
- 장기적으로는 메인 앱을 여는 방식보다, 사용자가 원래 보던 앱 흐름을 끊지 않는 공유 전용 저장 UX를 우선한다.
- 이상적인 형태는 공유 시트에서 Clippi를 눌렀을 때 즉시 작은 저장 UI가 뜨고, 카테고리 선택과 저장을 빠르게 끝낼 수 있는 구조다.
- 즉 iOS 기준으로는 Share Extension 기반 저장 UX를 장기 필수 기능으로 본다.
- 공유 저장 UX는 "저장 진입장벽 최소화"라는 제품 핵심 가치와 직접 연결된다.

## AI 적용 원칙
- 제목은 AI가 자동으로 덮어쓰지 않고 추천만 한다.
- 원문 제목은 항상 보존한다.
- 소분류는 기본적으로 추천값으로 제안한다.
- 소분류 자동 적용은 기존 소분류 재사용이 매우 확실한 경우에만 제한적으로 허용한다.
- 새 소분류는 자동 생성보다 사용자 선택을 우선한다.
- 카테고리는 추천이 애매하면 넣지 않고 미분류 상태로 둔다.
- 설명, 썸네일 같은 메타데이터는 보조 정보로 자동 반영한다.
- 초기에는 보수적으로 운영하고, 나중에 실제 사용자 데이터와 수정 패턴을 보며 기준을 보정한다.

## 지금 단계에서 절대 범위를 넓히지 말아야 할 항목
- TikTok, Instagram를 snackable viewer 흐름에 포함하는 것
- "콘텐츠 소비 앱" 방향으로 제품 정체성을 옮기는 것
- 복잡한 추천 피드, 알고리즘 피드, 소셜 기능 추가
- 무거운 상태관리 도입을 전제로 한 대규모 구조 변경
- 웹뷰 중심의 범용 인앱 브라우저를 제품 중심 기능으로 키우는 것
- 카테고리 체계 전체를 갈아엎는 대형 리팩터링

## 향후 고려 가능하지만 지금은 제외할 항목
- X/Twitter 인증 세션 유지
- 검색 강화
- 소분류 자동화 정확도 개선
- YouTube viewer의 뒤로가기 제스처 충돌 완화
- viewer 내 메타 정보 표현 개선
- iOS Share Extension 기반 공유 저장 UI
- 카테고리/소분류 편집 UX 정교화
- 북마크 편집, 정렬, 중복 관리 강화
- 친구/지인/팀과 북마크를 공유하고 함께 보는 협업 기능
- 북마크 모음집 단위 공유와 반응(좋아요/싫어요) 기능
- 북마크/모음집에 대한 짧은 코멘트 기반 소통 기능
- 초대 기반 소규모 그룹 공유 기능
- 긴 채팅형 대화 기능은 장기 고려 항목으로만 유지
- 북마크 기반 코디 추천
- 북마크 기반 운동 루틴 추천
- 북마크 기반 식단 추천
- 유료 요금제 기반 개인화 활용 기능

## 제품 원칙
- 북마크 앱의 본질을 먼저 지킨다.
- 무료 AI는 비용이 낮은 접근성 기능으로 적극 활용한다.
- 사용자가 북마크의 정체성으로 느끼는 값은 추천 중심으로 다룬다.
- 기능을 넓히기보다 저장 후 정리와 재소비의 완성도를 높인다.
- YouTube viewer는 중심 기능이 아니라 예외적 보조 UX로 유지한다.
- 장기적으로는 정리 앱을 넘어, 저장한 북마크를 실제 활용으로 연결하는 방향을 지향한다.
- 프리미엄 UX를 지향하되, 지금 단계에서는 작은 흐름 완성도를 우선한다.
