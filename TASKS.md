# Clippi 작업 목록

## 1. 소분류 선택 모드 안정화
목적: 소분류 `+` 진입 후 선택, 취소, 완료 흐름을 삭제 모드와 같은 수준으로 안정화한다.

예상 수정 파일
- `lib/screens/category_screen.dart`

주의사항
- 현재 삭제 모드와 충돌하지 않게 상태를 분리해야 한다.
- 이미 해당 소분류에 있는 북마크는 중복 선택되지 않게 유지한다.
- YouTube 진입, 일반 북마크 상세 진입을 깨지 않도록 선택 모드일 때만 탭 동작을 바꾼다.

## 2. 공유 저장 UX 방향 확정
목적: 공유하기에서 Clippi를 눌렀을 때 메인 앱 전체를 여는 대신, 사용자가 보던 앱 흐름을 최대한 끊지 않는 저장 UX를 제품 핵심 방향으로 확정한다.

예상 수정 파일
- `PRODUCT.md`
- `ROADMAP.md`
- `TASKS.md`

주의사항
- 지금 당장 구현 범위를 과하게 넓히지 않는다.
- 단순 Flutter 바텀시트 개선과 iOS Share Extension 기반 UX를 구분해서 본다.
- 공유 저장은 보조 기능이 아니라 핵심 저장 진입 경로라는 전제를 유지한다.

## 3. 무료 AI 첫인상 품질 정리
목적: AI 제목과 AI 소분류가 무료 버전의 차별점으로 느껴지도록 기본 품질을 끌어올린다.

예상 수정 파일
- `lib/screens/add_bookmark_sheet.dart`
- `lib/services/ai_service.dart`

주의사항
- API 비용을 크게 늘리는 방향으로 가면 안 된다.
- 무조건 자동 적용 확대보다 "처음 썼을 때 납득되는 결과"가 우선이다.

## 4. AI 적용 정책 구현
목적: 제목, 소분류, 카테고리, 메타데이터의 자동/추천 정책을 실제 저장 흐름에 반영한다.

예상 수정 파일
- `lib/screens/add_bookmark_sheet.dart`
- `lib/services/ai_service.dart`

주의사항
- 제목은 자동 덮어쓰기 금지, 추천만 유지해야 한다.
- 소분류 자동 적용은 기존 분류 재사용이 매우 확실할 때만 허용해야 한다.
- 카테고리 추천이 애매하면 미분류 상태를 유지해야 한다.
- 사용자가 쉽게 채택 또는 수정할 수 있는 UX가 필요하다.
- 초기에는 추천 기준을 보수적으로 운영하고, 이후 베타/실사용 데이터를 보며 조정해야 한다.

## 5. 카테고리 화면 소분류 헤더 UX 정리
목적: `분류 없음`, 수동 생성 소분류, 실제 북마크가 있는 소분류의 표현 규칙을 명확히 한다.

예상 수정 파일
- `lib/screens/category_screen.dart`

주의사항
- 현재 칩 구조를 크게 뒤엎지 않는다.
- 빈 소분류와 실제 항목이 있는 소분류의 시각 규칙을 너무 복잡하게 만들지 않는다.

## 6. CategoryScreen 리빌드/성능 점검
목적: 카테고리 화면 탐색 시 버벅임, 과도한 리빌드, 불필요한 상태 갱신을 줄인다.

예상 수정 파일
- `lib/screens/category_screen.dart`

주의사항
- 큰 리팩터링 없이 현재 `StatefulWidget + Isar stream` 구조를 유지한다.
- 체감 성능 개선이 목적이지 구조 교체가 목적이 아니다.

## 7. 북마크 재소비 유도 UX 점검
목적: 저장한 북마크를 다시 꺼내보게 만드는 흐름을 작은 개선 단위로 정리한다.

예상 수정 파일
- `lib/screens/home_screen.dart`
- `lib/screens/category_screen.dart`

주의사항
- 피드형 제품으로 확장하지 않는다.
- "다시 보기 쉽게" 만드는 것이지 "더 오래 붙잡기"가 목적이 아니다.

## 8. YouTube URL 판별 로직 정리
목적: YouTube 진입 조건과 Shorts 판별 기준을 한 곳에서 관리해 오판을 줄인다.

예상 수정 파일
- `lib/screens/category_screen.dart`
- `lib/screens/youtube_viewer_screen.dart`
- 필요 시 신규 유틸 파일 1개

주의사항
- 세로형 viewer는 보조 UX이며 YouTube 전용 범위를 유지한다.
- TikTok, Instagram는 절대 이 흐름에 넣지 않는다.

## 9. YouTube viewer 전환 체감 개선
목적: YouTube viewer를 핵심 기능처럼 키우지 않되, 현재 있는 보조 재소비 UX로서 거슬리지 않게 다듬는다.

예상 수정 파일
- `lib/screens/youtube_viewer_screen.dart`

주의사항
- 현재 재생 성공 플로우를 깨지 않게 작은 단계로 나눈다.
- 플레이어 패키지 교체 같은 큰 변경은 하지 않는다.

## 10. YouTube viewer 뒤로가기 충돌 완화
목적: 세로 스크롤 중 OS 뒤로가기 제스처나 의도치 않은 이탈을 줄인다.

예상 수정 파일
- `lib/screens/youtube_viewer_screen.dart`

주의사항
- 네비게이션 잠금을 과하게 넣어 답답한 UX가 되지 않게 한다.
- iOS/Android 모두에서 자연스러운 동작을 우선한다.

## 11. 메타데이터 실패 폴백 정리
목적: 메타데이터 수집 실패 시에도 저장 경험이 어색하지 않게 만든다.

예상 수정 파일
- `lib/screens/add_bookmark_sheet.dart`
- `lib/services/metadata_service.dart`

주의사항
- X/Twitter의 `fixupx.com` 처리 규칙은 유지한다.
- 실패 시 저장 자체가 막히지 않도록 해야 한다.

## 12. 중복 북마크 처리 UX 보강
목적: 같은 URL이 다시 저장될 때 덮어쓰기 동작이 사용자 입장에서 더 이해 가능하도록 만든다.

예상 수정 파일
- `lib/screens/add_bookmark_sheet.dart`
- `lib/services/isar_service.dart`

주의사항
- 현재 중복 방지 로직은 유지하되, 사용자 혼란만 줄이는 방향으로 접근한다.
- 데이터 구조 변경은 가능한 한 피한다.

## 13. 유료 활용 기능의 문제 정의 문서화
목적: 패션 코디, 운동 루틴, 식단 추천 같은 장기 유료 기능을 구현 전에 문제 정의 수준에서 정리한다.

예상 수정 파일
- `PRODUCT.md`
- `ROADMAP.md`
- `TASKS.md`

주의사항
- 지금 당장 기능 구현으로 넘어가지 않는다.
- 어떤 입력 데이터가 필요하고, 어떤 추천 결과가 가치가 있는지부터 정리한다.

## 14. 제품 방향과 handoff 문서 동기화
목적: 현재 제품 정의와 실제 코드 상태가 문서상으로도 어긋나지 않게 유지한다.

예상 수정 파일
- `HANDOFF_CODEX.md`
- `PRODUCT.md`
- `ROADMAP.md`
- `TASKS.md`

주의사항
- 문서가 구현을 과장하지 않도록 한다.
- viewer 범위와 제품 정체성을 일관되게 유지한다.

## 15. 협업/공유 기능 문제 정의 기록
목적: 친구, 지인, 팀과 북마크를 공유하고 함께 보는 기능을 장기 보류 아이템으로 정리한다.

예상 수정 파일
- `PRODUCT.md`
- 필요 시 `ROADMAP.md`

주의사항
- 지금 당장 구현하지 않는다.
- 개인 북마크 앱의 핵심 흐름을 흔들지 않도록 장기 검토 항목으로만 둔다.
- 단순 공유 링크, 모음집 공유, 반응(좋아요/싫어요), 짧은 코멘트, 채팅 기능을 한 번에 묶지 말고 단계적으로 분리해서 본다.
- 우선순위는 `공유 -> 반응 -> 짧은 코멘트 -> 초대 기반 소규모 그룹 -> 긴 채팅 고려` 순으로 둔다.
