# Clippi 로드맵

## 방향
로드맵은 기능 확장보다 현재 존재하는 저장, 정리, 재소비 흐름의 완성도를 높이는 순서로 잡는다.

특히 무료 AI는 초기 접근성과 차별점으로 강화하고, YouTube snackable viewer는 보조 재소비 UX로 유지하되 범위를 다른 플랫폼으로 넓히지 않는다.

## Phase 1
목표: 현재 북마크 저장, 무료 AI 정리, 소분류 재탐색 흐름을 안정화한다.

핵심 범위
- 카테고리 화면의 소분류 UX 정리
- 삭제/이동/추가 선택 모드 일관성 확보
- 공유로 들어온 링크 저장 흐름 안정화
- 공유 저장 UX의 장기 목표를 메인 앱 열기 방식이 아닌 공유 전용 저장 흐름으로 명확히 정의
- 중복 저장, 제목/메타데이터 반영 흐름 점검
- 무료 AI 제목/소분류 결과의 체감 품질 점검
- AI 자동 적용 정책 정리

완료 기준
- 카테고리 화면에서 소분류 관련 주요 동작이 충돌 없이 일관되게 동작한다.
- 저장된 북마크가 카테고리와 소분류 기준으로 다시 찾기 쉬운 상태를 유지한다.
- 저장 직후 AI 정리 결과가 "처음 써볼 만한 이유"로 느껴질 정도의 기본 품질을 갖는다.
- 제목은 추천 중심, 카테고리는 애매하면 미분류, 소분류는 보수적으로 자동 적용하는 정책이 정리된다.
- 초기 추천/자동 적용 결과를 이후 기준 보정에 활용할 수 있도록 운영 원칙이 정리된다.
- 치명적인 UI 버벅임이나 리빌드 문제 없이 기본 탐색이 안정적이다.
- 공유 저장 UX를 추후 Share Extension 기반으로 확장할지에 대한 제품 방향이 문서상 명확해진다.

## Phase 2
목표: 북마크 재소비 경험을 강화하되, 그중 YouTube viewer는 보조 UX로 다듬는다.

핵심 범위
- iOS Share Extension 기반 빠른 저장 UX 설계 및 구현 검토
- 다시 보기 쉬운 카테고리/소분류 흐름 보강
- 북마크 재방문을 유도하는 UX 보강
- Shorts와 일반 영상 처리 기준 정리
- 자동재생과 페이지 전환 체감 개선
- 뒤로가기 제스처 충돌 완화
- viewer 정보 레이어와 로딩 상태 개선

완료 기준
- 사용자가 저장한 북마크를 다시 꺼내보는 흐름이 더 자연스럽다.
- 공유하기에서 Clippi를 눌렀을 때 메인 앱 복귀 강제 없이 빠르게 저장할 수 있는 방향이 구체화되거나 구현된다.
- 유튜브 링크 진입 시 viewer 흐름이 깨지지 않는다.
- 자동 넘김, 세로 스크롤, 재생 전환이 눈에 띄게 덜 거슬린다.
- viewer 사용 중 실수로 이탈하거나 UX가 어색한 구간이 크게 줄어든다.

## Phase 3
목표: AI 기반 정리에서 AI 기반 활용으로 확장 가능한 제품 기반을 정리한다.

핵심 범위
- AI 요약 제목 품질 개선
- AI 소분류 추천 품질 개선
- 추천 결과의 적용 기준 정리
- 메타데이터 실패 시 폴백 경험 개선
- 유료 활용 기능의 문제 정의와 입력 데이터 구조 정리
- 패션/운동/레시피 같은 활용 시나리오를 제품 문서 수준에서 구체화

완료 기준
- 사용자가 수동 수정 없이도 제목/소분류가 자주 납득 가능한 수준으로 들어간다.
- AI 실패 시에도 저장 흐름이 불편하지 않다.
- 향후 유료 기능이 "정리 이후 활용"이라는 명확한 축으로 정의된다.

## 우선순위 원칙
- 1순위: 저장과 정리 흐름 안정화
- 2순위: 무료 AI의 초기 체감 가치 강화
- 3순위: 재소비 UX 보강
- 4순위: 장기 유료 활용 기능을 위한 설계
